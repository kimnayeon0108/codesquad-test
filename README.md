# codesquad-test    
## 3단계 루빅스 큐브 
- [참고 링크](https://cube3x3.com/%ED%81%90%EB%B8%8C%EB%A5%BC-%EB%A7%9E%EC%B6%94%EB%8A%94-%EB%B0%A9/#notation)를 참고해서 루빅스 큐브를 구현한다.
- 큐브는 W, B, G, Y, O, R의 6가지 색깔을 가지고 있다.
- 입력: 각 조작법을 한 줄로 입력받는다.
- 출력: 큐브의 6면을 펼친 상태로 출력한다.
- Q를 입력받으면 프로그램을 종료하고, 조작 받은 명령의 갯수를 출력시킨다.

## 큐브의 초기상태

```
                B B B  
                B B B
                B B B

 W W W     O O O     G G G     Y Y Y 
 W W W     O O O     G G G     Y Y Y 
 W W W     O O O     G G G     Y Y Y 
 
                R R R 
                R R R 
                R R R 
```

## 프로그램 예시

```
(초기 상태 출력)

CUBE> FRR'U2R

F
(큐브상태)

R
(큐브상태)

...

R
(큐브상태)

CUBE> Q
경과시간: 00:31 //추가 구현 항목
조작갯수: 6
이용해주셔서 감사합니다. 뚜뚜뚜.
```

## 추가 구현 기능

- 프로그램 종료 시 경과 시간 출력
- 큐브의 무작위 섞기 가능
- 모든 면을 맞추면 축하 메시지와 함께 프로그램을 자동 종료

## 요구사항

- 가능한 한 커밋을 자주 하고 구현의 의미가 명확하게 전달되도록 커밋 메시지를 작성할 것
- 함수나 메소드는 한 번에 한 가지 일을 하고 가능하면 20줄이 넘지 않도록 구현한다.
- 함수나 메소드의 들여쓰기를 가능하면 적게(3단계까지만) 할 수 있도록 노력해 본다.

## 구현

- 2단계보다 2개의 위치(F면과 D면)가 추가로 생겼다. (2단계의 B와 별개로 3단계에서 B는 아래가 아닌 Back 으로 간주)
- 2차열 배열을 6개 만들까, 큐브의 각각 면 마다 클래스를 만들어야 하나 고민하다가 전체 큐브 면을 출력하기에 3차열 배열이 제일 알맞을 것 같다고 판단하였다.

## 함수의 역할 및 동작 설명

- RubiksCube의 생성자에 함수를 호출하도록 하여 객체 생성과 동시에 프로그램이 동작하도록 했다.

### getCube()

- for문안에 cube의 페이지에 해당되는 수 만큼 getCube()를 실행하여 큐브 배열에 값 할당

### addElement()

- 큐브 배열에 요소를 채우는 메소드
- 매개변수로 받은 int z 값을 큐브의 페이지로 지정하고, 매개변수 char a 를 요소로 할당

### printCube()

- 정육면체의 큐브를 평면도로 출력하는 메소드
- 맨 위에 B면, 그 아래에 나란히 W, O, G, Y 면, 맨 아래에 R면이 출력되게 함

### printPage()

- 매개변수에 해당하는 페이지를 출력하는 메소드
- 한 줄에 한 페이지만 출력되는 페이지를 위한 메소드 (B, R)

### printFourP()

- 매개변수에 해당하는 페이지를 평면도 조건에 맞게 출력하는 메소드
- 한 줄에 4페이지가 나란히 출력되는 페이지를 위한 메소드 (W, O, G, Y)

### start()

- 사용자에게 입력값을 받고, 프로그램을 본격적으로 실행시키는 메소드

### getInput()

- 사용자에게 조작 명령어를 입력 받는다.
- Q를 입력할 시 매개변수로 받은 조작 개수(num)를 출력하고 프로그램을 종료한다.

### getArrSize()

- 입력 받은 명령어를 alphaArr 배열에 넣기 위해 입력받은 값(input) 중 알파벳의 개수를 세고, 그 개수를 alphaArr의 크기로 설정하는 메소드

### getAlphaArr()

- 입력 받은 값(input)을 alphaArr의 값으로 넣어주는 메소드
- input의 글자 하나하나를 돌면서 마지막 글자가 아닌 경우에 다음글자에 " ' "나 "2"의 유무를 확인하고 조건에 맞게 값을 넣는다.

### move()

- alphaArr 배열의 요소마다 해당하는 메소드를 실행시킨다.
- 요소에 "2"가 포함되어 있을 시, 해당 메소드를 두번 실행
- 각 요소 실행시 마다 큐브 출력

### moveByInput()

- alphaArr의 요소를 매개변수로 받아서 해당하는 메소드 호출하는 역할

### moveFtoL()

- 큐브의 앞쪽 면을 시계방향(왼쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 1면 2열 → temp값에 담기
    2. 5면의 0행 → 1면의 2열로
    3. 3면의 0열 → 5면의 0행으로
    4. 0면의 2행 → 3면의 0열로
    5. temp → 0면의 2행으로

### moveFtoR()

- 큐브의 앞쪽 면을 반시계방향(오른쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 1면의 2열 → temp 값
    2. 0면의 2행 → 1면의 2열로
    3. 3면의 0열 → 0면의 2행으로
    4. 5면의 0행 → 3면의 0열로
    5. 5면의 0행 → temp

### moveRtoL()

- 큐브의 오른쪽 면을 시계방향(왼쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 2면의 2열 temp 값에 담기
    2. 5면의 2열 → 2면의 2열로
    3. 4면의 0열 → 5면의 2열로
    4. 0면의 2열 → 4면의 0열로
    5. temp 값 → 0면의 2열로

### moveRtoR()

- 큐브의 오른쪽 면을 반시계방향(오른쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 2면의 2열 temp 값에 담기
    2. 0면의 2열 → 2면의 2열로
    3. 4면의 0열 → 0면의 2열로
    4. 5면의 2열 → 4면의 0열로
    5. temp → 5면의 2열로 

### moveUtoL()

- 큐브의 위쪽 면을 시계방향(왼쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 1면의 0행 temp 값에 담기
    2. 2면의 0행 → 1면의 0행으로
    3. 3면의 0행 → 2면의 0행으로
    4. 4면의 0행 → 3면의 0행으로
    5. temp → 4면의 0행으로

### moveUtoR()

- 큐브의 위쪽 면을 반시계방향(오른쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 1면의 0행 temp 값에 담기
    2. 4면의 0행 →  1면의 0행으로
    3. 3면의 0행 → 4면의 0행으로
    4. 2면의 0행 → 3면의 0행으로
    5. temp → 2면의 0행으로

### moveBtoR()

- 큐브의 뒤쪽 면을 시계방향(오른쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 1면의 0열 temp 에 담기
    2. 0면의 0행 → 1면의 0열
    3. 3면의 2열 → 0면의 0행
    4. 5면의 2행 → 3면의 2열
    5. temp → 5면의 2행

### moveBtoL()

- 큐브의 뒤쪽 면을 반시계방향(왼쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 1면의 0열 temp 에 담기
    2. 5면의 2행 → 1면의 0열
    3. 3면의 2열 → 5면의 2행
    4. 0면의 0행 → 3면의 2열
    5. temp → 0면의 0행

### moveLtoR()

- 큐브의 왼쪽 면을 시계방향(오른쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 2면의 0열 temp 에 담기
    2. 0면의 0열 → 2면의 0열로
    3. 4면의 2열 → 0면의 0열로
    4. 5면의 0열 → 4면의 2열로
    5. temp → 5면의 0열

### moveLtoL()

- 큐브의 왼쪽 면을 반시계방향(왼쪽)으로 옮기는 메소드
- 움직임 변화 순서
    1. 2면의 0열 → temp
    2. 5면의 0열 → 2면의 0열
    3. 4면의 2열 → 5면의 0열
    4. 0면의 0열 → 4면의 2열
    5. temp → 0면의 0열

### moveDtoR()

- 큐브의 아래쪽 면을 시계방향(오른쪽)으로 옮기는 메소드
- 움직임 변화순서
    1. 1면의 2행 → temp
    2. 4면의 2행 → 1면의 2행
    3. 3면의 2행 → 4면의 2행
    4. 2면의 2행 → 3면의 2행
    5. temp → 2면의 2행

### moveDtoL()

- 큐브의 아래쪽 면을 반시계방향(왼쪽)으로 옮기는 메소드
- 움직임 변화순서
    1. 1면의 2행 → temp
    2. 2면의 2행 → 1면의 2행
    3. 3면의 2행 → 2면의 2행
    4. 4면의 2행 → 3면의 2행
    5. temp → 4면의 2행

### turnClock()

- 매개변수의 값에 해당하는 면이 시계방향으로 도는 메소드
- 각 행에 해당하는 값을 temp1, temp2, temp3에 넣는다.
- 움직이는 규칙에 맞도록 temp 값을 각 열에 담는다.
    1. 0행 → 2열로
    2. 1행 → 1열로
    3. 2행 → 0열로 

### turnAntiClock()

- 매개변수의 값에 해당하는 면이 반시계방향으로 도는 메소드
- turnClock() 과 마찬가지로 각 행의 값을 temp1, temp2, temp3 에 넣고, 움직이는 규칙에 맞도록 temp 값을 각 열에 담는다.
    1. 0행 → 0열로
    2. 1행 → 1열로
    3. 2행 → 2열로
